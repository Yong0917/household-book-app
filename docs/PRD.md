# 가계부 앱 (moneylog-app) MVP PRD

## 핵심 정보

**목적**: 개인 수입과 지출을 날짜별로 기록하고 카테고리별 통계로 소비 패턴을 파악한다
**사용자**: 스마트폰으로 일상 수입/지출을 간편하게 관리하고 싶은 1인 가계부 사용자

## 개발 단계

백엔드 및 API 연동 전 UI/UX를 먼저 검증하는 단계적 접근 방식을 채택한다.

### Phase 1: UI 프로토타입

**목표:** 실제 기능 없이 화면 구조와 내비게이션 플로우를 구현하여 사용자 경험을 사전 검증한다.

- Mobile First 기반 반응형 레이아웃 구현
- 하단 탭 내비게이션, 바텀 시트 등 핵심 UI 컴포넌트 구현
- 정적 화면 기반으로 페이지 간 이동 및 사용자 플로우 검증
- 실제 데이터 연결 없이 화면 구조 중심으로 구성

### Phase 2: 더미 데이터 기반 개발

**목표:** Mock 데이터를 활용해 API 연결 없이 화면 동작과 인터랙션을 구현하고 UX를 사전 검증한다.

- 하드코딩된 Mock 데이터로 거래 목록, 통계, 카테고리 등 화면 동작 구현
- 상태 관리(useState 등) 중심으로 CRUD 인터랙션 구현
- 달력 날짜 선택, 바텀 시트 열기/닫기 등 UX 인터랙션 완성
- 실제 API 연결 전 인터랙션 패턴 확정

### Phase 3: 백엔드 연동

**목표:** 검증된 UI/UX를 기반으로 Supabase DB 스키마 생성, Server Actions 구현, 실제 데이터를 연동한다.

- Supabase 테이블 생성 및 RLS 정책 적용
- Server Actions로 거래·카테고리·자산 CRUD 구현
- Mock 데이터를 실제 API 호출로 교체
- 인증 흐름 연동 및 데이터 보안 검증

## 사용자 여정

```
1. 로그인 페이지 (이미 구현)
   - 이메일/비밀번호 입력
   ↓ 로그인 성공

2. 가계부 일일 보기 (기본 화면, /ledger/daily)
   - 오늘 날짜 수입/지출 목록 표시
   - 하단 탭: 가계부 | 통계 | 설정
   ↓ 사용자 선택 분기

   [FAB + 버튼 탭] → 거래 등록 바텀 시트 열림
        ↓ 저장 완료 → 일일 보기로 돌아옴

   [거래 항목 탭] → 거래 수정/삭제 바텀 시트 열림
        ↓ 저장/삭제 완료 → 일일 보기로 돌아옴

   [달력 보기 탭 탭] → 달력 보기 (/ledger/calendar)
        ↓ 날짜 셀 탭
        → 해당일 거래 목록 표시 (달력 하단)

   [하단 탭: 통계] → 통계 페이지 (/statistics/income 기본)
        ├─ 수입 탭 → 수입 통계 도넛 차트
        └─ 지출 탭 → 지출 통계 도넛 차트

   [하단 탭: 설정] → 설정 페이지 (/settings)
        ├─ 분류 관리 → 카테고리 CRUD (/settings/categories)
        └─ 자산 관리 → 자산 CRUD (/settings/assets)
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | 거래 등록 | 날짜·시간, 금액, 분류, 자산, 내용을 입력해 수입/지출 거래를 저장 | 가계부의 핵심 데이터 입력 수단 | 가계부 일일 보기, 달력 보기 |
| **F002** | 거래 수정/삭제 | 기존 거래 항목을 탭해 내용 수정 또는 삭제 | 오기입 수정 불가 시 서비스 이탈 발생 | 가계부 일일 보기, 달력 보기 |
| **F003** | 가계부 일일 보기 | 선택 날짜의 수입/지출 목록과 일별 합계를 시간순으로 표시 | 사용자가 가장 자주 접근하는 핵심 화면 | 가계부 일일 보기 |
| **F004** | 달력 보기 | 월 단위 달력에서 날짜별 수입(파랑)/지출(빨강) 합계 금액 표시 | 월 전체 흐름 파악을 위한 필수 뷰 | 달력 보기 |
| **F005** | 수입 통계 | 선택 월의 카테고리별 수입 합계를 도넛 차트와 목록으로 표시 | 소비 패턴 파악을 위한 핵심 인사이트 | 통계 페이지 |
| **F006** | 지출 통계 | 선택 월의 카테고리별 지출 합계를 도넛 차트와 목록으로 표시 | 소비 패턴 파악을 위한 핵심 인사이트 | 통계 페이지 |

## 데이터 모델

### profiles (사용자 프로필)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 (auth.users와 동일) | UUID |
| display_name | 표시 이름 | TEXT |
| created_at | 생성 일시 | TIMESTAMPTZ |

### categories (거래 분류)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| user_id | 소유 사용자 | → auth.users.id |
| name | 카테고리 이름 (예: 식비, 급여) | TEXT |
| type | 수입/지출 구분 | TEXT (income / expense) |
| icon | 아이콘 식별자 | TEXT (lucide-react 아이콘명) |
| color | 표시 색상 | TEXT (hex, 예: #FF5733) |
| is_default | 기본 프리셋 여부 | BOOLEAN |
| sort_order | 목록 정렬 순서 | INT |
| created_at | 생성 일시 | TIMESTAMPTZ |

**기본 seed 데이터:**
- 수입: 급여, 용돈, 부업, 금융, 기타
- 지출: 식비, 교통, 주거, 의류, 의료, 문화, 교육, 미용, 경조사, 기타

### assets (자산)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| user_id | 소유 사용자 | → auth.users.id |
| name | 자산 이름 (예: 현금, 신한은행) | TEXT |
| type | 자산 유형 | TEXT (cash / bank / card / other) |
| icon | 아이콘 식별자 | TEXT (lucide-react 아이콘명) |
| color | 표시 색상 | TEXT (hex) |
| is_default | 기본 프리셋 여부 | BOOLEAN |
| sort_order | 목록 정렬 순서 | INT |
| created_at | 생성 일시 | TIMESTAMPTZ |

**기본 seed 데이터:** 현금, 은행계좌, 신용카드

### transactions (거래 내역)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| user_id | 소유 사용자 | → auth.users.id |
| type | 수입/지출 구분 | TEXT (income / expense) |
| amount | 금액 (원 단위, 양수만 저장) | BIGINT |
| category_id | 거래 분류 | → categories.id |
| asset_id | 사용 자산 | → assets.id |
| description | 내용/메모 | TEXT (nullable) |
| transaction_at | 거래 발생 일시 (사용자 입력) | TIMESTAMPTZ |
| created_at | 레코드 생성 일시 | TIMESTAMPTZ |
| updated_at | 레코드 수정 일시 | TIMESTAMPTZ |

**RLS 정책:** 모든 테이블에서 `user_id = auth.uid()` 기반으로 행 단위 보안 적용

## 인증 흐름 (기존 구현 유지)

```
미인증 사용자 접근 (/ledger/*, /statistics/*, /settings/*)
  ↓ proxy.ts → lib/supabase/proxy.ts updateSession()
  → /auth/login 으로 리디렉션

로그인 성공
  → /ledger/daily 로 리디렉션 (기본 화면)
```

**변경 사항:** 기존 `/protected` 리디렉션을 `/ledger/daily`로 변경 필요
- `lib/supabase/proxy.ts`: 인증 성공 후 `/ledger/daily`로 리디렉션
- `app/auth/login/page.tsx` 또는 login Server Action: 성공 시 `/ledger/daily`로 이동
